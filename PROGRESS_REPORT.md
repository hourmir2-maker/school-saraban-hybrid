## ⚙️ 2. การปรับปรุงซอร์สโค้ดและฟีเจอร์ในวันนี้

### 1. ระบบโต้ตอบสั่งการหนังสือแบบตอบโต้บน Telegram (V3)
* พัฒนาระบบรับ Callback Query ใน `api/telegram-webhook.ts` เพื่อประมวลผลปุ่มกดสั่งการ ผอ. ทันที
* พัฒนาระบบประทับตรา PDF อัตโนมัติในฝั่ง Serverless ฟังก์ชันเมื่อมีการกดยืนยัน (ฝังลายเซ็น ผอ., ข้อความสั่งการ, วันที่ย่อเลขไทย และส่งขึ้น Google Drive ผ่าน GAS)
* **แก้ไขบั๊ก `BUTTON_DATA_INVALID`**: แก้ไขข้อจำกัดของ Telegram API ที่ห้ามส่งข้อมูล Callback Data เกิน 64 Bytes โดยบีบอัดโค้ดปุ่มกดลงเหลือ 20-30 bytes (ใช้ Shortcodes เช่น `ins=1`, `ins=2`, `bc_grp`) และใช้ตารางจำเซสชันฝั่ง Supabase ในการประกอบร่างข้อมูลแทน

### 2. ระบบแยกกลุ่มแจ้งเตือน (Central vs Proposal Group Routing)
* ปรับปรุงหน้า **ตั้งค่าโรงเรียน (Settings.tsx)** ให้ธุรการระบุรหัสกลุ่ม Telegram แยกกันได้ 2 กลุ่ม:
  1. **กลุ่มแจ้งเตือนส่วนกลาง / PR** (สำหรับแชร์เอกสารที่เซ็นแล้วให้ครูทุกคนทราบ)
  2. **กลุ่มเสนอหนังสือเกษียณ** (สำหรับส่งให้ ผอ. และธุรการพิจารณาลงนามสั่งการ)
* ใช้เทคนิคจัดเก็บรวมลงในคอลัมน์เดียวด้วยสัญลักษณ์คั่น Pipe (`|`) เพื่อให้ย้อนกลับไปใช้ตารางเดิมได้และไม่ทำให้ฐานข้อมูลพัง
* พัฒนา Fallback อัตโนมัติให้ส่งหาแชทส่วนตัวของ ผอ. หากยังไม่มีการตั้งค่ากลุ่มในระบบ

### 3. ระบบเสนอหนังสือพร้อมกันหลายฉบับแบบกลุ่ม (Telegram Bulk Proposals)
* ปรับแก้ฟังก์ชัน `handleBulkPropose` ในหน้าจอลงรับหนังสือ (`IncomingDocs.tsx`) ให้ส่งแจ้งเตือนเข้า Telegram พร้อมกันกับ LINE
* ออกแบบข้อความเสนอหนังสือแบบกลุ่มจัดรวมเป็นรายการเดียวเพื่อความเรียบร้อย พร้อมแนบปุ่มเกษียณสั่งการแยกเล่มตามไอดีจริงที่มีขนาดสอดคล้องกับข้อจำกัด 64 Bytes ของ Telegram
